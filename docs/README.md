# Achika Site — Documentation

This directory contains project documentation for `anapadiuk/achika-site`.

## Recommended reading order

1. **AI context and copy rules:** `../AI_CONTEXT.md` — product, ICP, tone, positioning decisions. Read first when working on content or copy.
2. **Marketing decisions:** `docs/marketing/` — what was decided and why, session by session.
3. **Business context:** `docs/archive/` — investor reviews and market assessment. Reference only, not directive.

## Directory structure

### `docs/marketing/`
Copy and positioning decisions: tone guidelines, ICP, key messaging choices, and what was changed and why.

### `docs/archive/`
Historical documents: investor reviews, superseded decisions. Kept for context and traceability.

## Rules

- The site HTML files are the source of truth for what is live.
- Docs explain *why* decisions were made — not just what.
- `AI_CONTEXT.md` must stay current: update it when positioning or tone decisions change.
- `docs/marketing/` is append-friendly: add a new dated file per session, don't overwrite old ones.
